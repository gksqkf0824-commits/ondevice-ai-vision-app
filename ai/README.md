# AI

## 개요
이 디렉토리는 프로젝트의 AI 파트를 위한 작업 공간입니다.

주요 작업 범위는 다음과 같습니다.

- 객체 탐지 모델 개발
- 데이터셋 전처리 및 병합
- 학습용 데이터 구성
- 모델 학습 및 평가
- 모바일 배포용 모델 export
- AI 관련 문서 정리

---

## 목표
본 프로젝트의 AI 파트 목표는 다음과 같습니다.

- 온디바이스 환경에서 동작 가능한 객체 탐지 모델 개발
- 시각약자 보조 서비스를 위한 클래스 설계 및 데이터셋 구성
- 모바일 파트에서 사용할 수 있는 형식으로 모델 산출물 전달

---

## 디렉토리 구조
예시 구조는 아래와 같습니다.

ai/
├─ README.md
├─ requirements.txt
├─ configs/
├─ scripts/
│  ├─ preprocess/
│  ├─ train/
│  ├─ eval/
│  └─ export/
├─ data/
│  ├─ raw/
│  ├─ interim/
│  ├─ processed/
│  └─ yolo/
├─ models/
└─ docs/

각 폴더의 역할은 다음과 같습니다.

- configs/ : 클래스 매핑, 학습 설정, 경로 예시 등 설정 파일
- scripts/preprocess/ : 데이터셋 스캔, 변환, 병합, YOLO 변환 스크립트
- scripts/train/ : 학습 실행 스크립트
- scripts/eval/ : 평가 및 분석 스크립트
- scripts/export/ : ONNX/OpenVINO 등 export 관련 스크립트
- data/ : 원본/중간/최종 데이터 구조 관리용 폴더
- models/ : 체크포인트 및 export 결과 관리용 폴더
- docs/ : 전처리, 학습, 인수인계 문서

## 작업 흐름
현재 AI 파트의 기본 작업 흐름은 아래와 같습니다.

1. 데이터셋 수집
2. 데이터셋 구조 및 클래스 스캔
3. 중간산출물 규격으로 변환
4. 여러 데이터셋 병합
5. YOLO 학습용 형식으로 변환
6. 모델 학습
7. 평가 및 오류 분석
8. 모바일 전달용 모델 export

## 업로드 금지
다음 항목은 GitHub에 업로드하지 않습니다.

- 원본 데이터셋
- 중간/최종 데이터셋 파일
- 모델 가중치 파일 (.pt, .pth, .onnx 등)
- 학습 결과물
- 개인 로컬 환경 파일

## 브랜치 및 작업 원칙
- AI 관련 코드는 ai/ 디렉토리 내부에서만 관리합니다.
- AI 작업은 ai 브랜치에서 진행합니다.
- main 브랜치에는 직접 push 하지 않습니다.
